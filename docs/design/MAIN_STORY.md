# 《算界旅人》——RPG 冒险游戏剧情设定文档（草案）

## 一、世界观总设定

在名为「算界」的平行世界中，一切文明都围绕「计算之火」发展。  
这团火种会在每一次技术突破时被点燃得更耀眼，也会在错误的道路上逐渐黯淡。

你扮演的主角是被称为「调试者（Debugger）」的旅人，被时间之河选中，可以穿梭各个时代：
- 机械计算时代
- 电子管时代
- 集成电路时代
- 半导体与微处理器时代
- 网络时代
- 算力时代

每个时代都有一个象征性的 **时代 Boss** 和成群的 **NPC 科学家 / 工程师**。  
与此同时，世界中还潜藏着两条关键支线：
- **编程语言发展史**（Language Branch）
- **操作系统发展史**（OS Branch）

玩家在主线与两条支线中的选择，将最终汇聚成四种结局：  
`BAD_END / NORMAL_END / HAPPY_END / TRUE_END`。

现实世界中每一次技术突破，在游戏中都具象化为 **装备 / 技能 / 道具**。

---

## 二、主角与核心系统设定

### 2.1 主角背景

- 名字：玩家自定，设定代号为「调试者」。
- 身份：被「时间编译器」选中的旅人，可以读取和修补历史中的「错误」。
- 初始能力：
  - 基础战斗（近战 + 远程简易工具）
  - 「解析之眼」：能看见技术节点（以符文或电路图形式呈现）
  - 限制级时间回溯：只能回溯到每个时代的特定「检查点」

### 2.2 技术成果 → 装备的规则

- **硬件类突破** → 武器 / 防具 / 被动技能
  - 继电器 → 「继电打击手套」（攻击时有几率触发连击）
  - 晶体管 → 「晶体之盾」（在高频攻击时提高防御）
  - 集成电路 → 「多路协同头盔」（提升技能冷却回复速度）
- **软件 / 理论类突破** → 主动技能 / 被动加成
  - 图灵机理论 → 技能「通用模拟」，可短时间复制敌人某个技能
  - UNIX 哲学 → 被动「一件事，做好」，提升专精技能威力

## 三、时代分章主线剧情

### 第零章：序章——时间编译器与破碎的算界

- 场景：一片漂浮着无数电路碎片与机械齿轮的「中立空间」。
- 关键 NPC：
  - 「时间编译器」：无形的声音，引导者。
- 剧情：
  1. 主角在一片空白空间中醒来，手里握着一块破碎的「计算之核」。
  2. 时间编译器告知：算界的历史被某种「未知错误」污染，时间线出现分叉和崩塌。
  3. 玩家需要依次进入各个时代，修复关键的技术节点，维持技术发展走向「健康」的轨道。
  4. 玩家得到第一件武器「算盘指环」（基础攻击 +1）和技能「解析之眼」。

---

### 第一章：机械计算时代 —「齿轮与蒸汽的乐章」

- 时间氛围：蒸汽朋克风格城市「分析城」，高耸的差分引擎塔、穿行的蒸汽列车、街道上随处可见的手摇计算器摊位与算盘行会。
- 关键 NPC（概要）：
  - 查尔斯·巴贝奇：差分引擎塔的总设计师，主线导师型 NPC，与资金和政治压力斗争。
  - 艾达·洛夫莱斯：将计算视为“可书写的乐谱”的天才贵族，编程语言支线的起点。
  - 「手动计算公会」长老们：代表旧有手工计算体系的守旧势力，其中一人后来与魔像融合成为 Boss。
  - 各类工匠 NPC：齿轮匠、电报匠、绘图员等，为玩家提供装备与支线任务。
- 主要剧情节点（概要）：
  1. 主角坠落在「分析城」外围的齿轮工坊区，发现差分引擎塔建设因骚乱而停工。
  2. 巴贝奇在政务大厅被保守派围攻，质疑其项目“浪费金钱、威胁手算者生计”，玩家第一次介入调停和战斗。
  3. 城市各处发生针对机械设施的破坏事件（火车转辙失灵、机械时钟集体停摆），玩家逐步查出是手动计算公会中的激进派所为。
  4. 玩家协助修复关键机械节点（齿轮传动、十进制拨盘、电报继电器原型），这些修复过程也为玩家解锁一批时代装备原型。
  5. 与艾达多次对话、协助她完成“计算乐谱”，在战斗与解谜中初次引入“指令序列”“循环”等概念，为编程语言支线埋下伏笔。
  6. 紧张局势在「算盘广场」的公开辩论中爆发为暴力冲突，激进长老借助被污染的算界之火召唤出机械魔像，成为本章 Boss。
- 时代 Boss：
  - 「手动计算公会首领」+ 被不稳定的机械魔像融合的狂暴形态，战斗中会召唤“手算阴影”不断复写错误数据。
- Boss 战象征：
  - 代表对技术的恐惧与排斥，以及因害怕被取代而选择破坏未来的冲动。
- 通关奖励（概要）：
  - 装备「差分之刃」：基础攻击提升，对“重复型敌人/召唤物”伤害更高。
  - 装备「齿轮背带」与「游标卡尺护腕」等机械时代防具，提升精度与格挡能力。
  - 若完成艾达支线：获得被动技能「算法草稿」（解锁编程语言支线第一节点，并在后续时代解锁更高阶“语言流派”）。

### 第二章：电子管时代 —「真空中的光芒」

- 场景：巨大机房、闪烁的电子管墙壁、轰鸣的电源室。
- 关键 NPC：
  - 阿兰·图灵
  - 以「Colossus 机组」「ENIAC 管理者」为原型的工程师 NPC
- 主要剧情节点：
  1. 主角抵达战时机密基地，巨型计算机正在为战争提供密码破译与弹道计算。
  2. 图灵正在设计一种抽象的「通用机」思想，却遭到周围人“不实用”的质疑。
  3. 玩家一边协助进行密码破译任务（遭遇战 + 解密小游戏），一边见证「通用计算机思想」形成。
  4. 同时也会看到技术被战争利用的残酷一面。
- 时代 Boss：
  - 「战争计算指挥核心」：一台被军方强化、只追求效率与杀伤输出的失控机。
- Boss 战象征：
  - 技术被扭曲用于极端破坏，而忽视人的价值。
- 通关奖励：
  - 技能「通用模拟」：短时间复制敌人一种技能。
  - 若与图灵保持高好感：解锁剧情，获得被动「逻辑严谨」提升技能命中/解谜能力。

---

### 第三章：集成电路时代 —「微缩奇迹」

- 场景：洁白的实验室、硅片工厂、线路如城市般密集的芯片平原（视觉拟人化）。
- 关键 NPC：
  - 杰克·基尔比
  - 罗伯特·诺伊斯
- 主要剧情节点：
  1. 主角进入「硅之谷」雏形世界，工程师们在尝试把大量元件集成在一块小小的芯片上。
  2. 玩家帮助解决「热量过高」「布线混乱」「成本过高」等具象化成怪物的难题。
  3. 城中有两个阵营：分立元件守旧派 vs 集成派，玩家需要在冲突中调和。
- 时代 Boss：
  - 「过载巨兽」：由无数分立元器件堆砌而成的怪物，象征低效、庞大、难以维护的旧时代。
- 通关奖励：
  - 装备「集成之冠」（减少技能冷却，提高同时进行任务的能力）。
  - 解锁被动「摩尔预言」：随着时间推进，角色属性周期性小幅提升（象征摩尔定律）。

---

### 第四章：半导体与微处理器时代 —「硅上帝脑」

- 场景：现代工厂、高洁净度无尘室、显微视角的「晶体管之城」。
- 关键 NPC：
  - 戈登·摩尔
  - 多位以 Intel / AMD 工程师为原型的虚构人物
- 主要剧情节点：
  1. 微处理器诞生，计算能力可以被放进每一台终端设备中。
  2. 玩家见证从大型机 → 小型机 → 个人计算机的权力下放过程。
  3. 城市出现「算力寡头」集团，试图垄断处理器设计与产能。
- 时代 Boss：
  - 「单极垄断核心」：吞噬了其他芯片设计的怪物，只运行自家指令集。
- 通关奖励：
  - 装备「多核护腕」：在战斗中可以召唤一个“影分身”进行辅助攻击。
  - 若在本章触发 OS 支线节点：解锁「操作系统内核外观」装备（对系统类敌人抗性提高）。

---

### 第五章：网络时代 —「万物互联的罗网」

- 场景：数据之城、光纤高速路、节点林立的「网络平原」。
- 关键 NPC：
  - 以 Vint Cerf、Tim Berners-Lee、Radia Perlman 等人为原型的虚构角色
  - 开源社区代表、黑客组织成员等
- 主要剧情节点：
  1. 玩家抵达「万维城」，这里信息高度自由流动。
  2. 玩家在「协议议会」任务中参与制定标准（象征 TCP/IP、HTTP、路由协议）。
  3. 但随着连接变得无处不在，「信息噪声」「恶意软件」「隐私泄漏」也化为具体敌人出现。
- 时代 Boss：
  - 「混沌路由者」：会随机改变战场规则（延迟、掉包、重定向攻击）。
- 通关奖励：
  - 技能「分布式同步」：战斗中可与友军 / 召唤物共享 Buff 状态。
  - 若完成多项网络正向选择：获得被动「网络中立」，减少来自高强度群攻的伤害。

---

### 第六章：算力时代 —「深度学习的迷雾」

- 场景：超算中心、数据湖、由矩阵构成的「高维平原」。
- 关键 NPC：
  - 综合象征 AI 研究者、芯片厂、云计算平台的多位虚构人物
- 主要剧情节点：
  1. 算界进入「算力为尊」的时代，一切问题似乎都可以交给黑盒模型处理。
  2. 玩家见到许多城市为了追求“智能化”而牺牲透明度、隐私与可解释性。
  3. 深处出现名为「推理之渊」的区域，所有决策被一个巨大的黑盒做出。
  4. 玩家必须决定：是让算力继续无限扩张，还是重构算界的价值观？
- 终极 Boss：
  - 「黑盒神谕」：一个不断学习玩家战斗模式的敌人，会进化策略。
- Boss 战象征：
  - 盲目的技术崇拜 vs 负责任的技术使用。

## 四、两条重要支线剧情

### 4.1 编程语言发展史支线（Language Branch）

- 起点：第一章中与艾达·洛夫莱斯深入对话。
- 关键节点：
  1. 第一章：艾达的「算法乐谱」 → 道具「算法手记」，提升解谜能力。
  2. 第二章：接触早期汇编与机器码，获得技能「底层把握」。
  3. 第三章：在集成电路时代，遇到类 Fortran/C 的语言设计者，任务包括“优化数值计算”、“抽象控制结构”。
  4. 第四章：高级语言百花齐放（C、C++、Java…），玩家可以选择不同风格的“编程流派天赋”（偏向底层/抽象/安全性等）。
  5. 第五章：脚本语言与 Web 语言崛起（Python、JavaScript 等），支线任务“快速构建工具支援主线”。
  6. 第六章：领域专用语言 / 模型描述语言，玩家可解锁最终技能「元语言重构」，允许在最终战中“修改战斗规则的一小部分”。
- 对结局影响：
  - 若编程语言支线完成度高，最终能在 TRUE_END 中以“重写系统规则”的方式脱离单一时间线束缚。

### 4.2 操作系统发展史支线（OS Branch）

- 起点：第三章中，工程师抱怨“程序越来越多，管理太混乱”。
- 关键节点：
  1. 第三章：早期批处理系统任务，玩家帮助“排队作业”、“调度任务”。
  2. 第四章：遇见以 UNIX 传统为原型的 NPC（简化为虚构人物或组合形象），任务是“用小工具拼接出强大工作流”。
  3. 第五章：图形界面 vs 命令行阵营冲突，玩家可调和平衡“易用性”和“可控性”。
  4. 第六章：分布式操作系统与云原生，任务包括“在多节点之间安全迁移任务实例”。
- 对结局影响：
  - OS 支线完成度高，可在最终 Boss 战中调用「资源调度特权」，极大降低战斗难度，并开启对 TRUE_END 的一部分条件。

## 五、结局设计

### BAD_END ——「算界崩坏」

- 条件（示例）：
  - 多次选择短视决策：例如只追求算力、不顾隐私与伦理。
  - 两条支线推进度极低，关键人物被忽视甚至死亡。
- 表现：
  - 算界被失控的黑盒系统接管，人类沦为“统计噪声”。
  - 主角被时间编译器强制弹出，世界编译失败，你看到的是“错误日志式”结尾。

### NORMAL_END ——「不完美的前进」

- 条件：
  - 主线全部完成，但至少一条支线推进一般或较低。
- 表现：
  - 算界整体保持向前发展，但留下明显隐患与遗憾。
  - 个别时代的决策导致局部区域“技术负债”极高。
  - 时间编译器评价为“可运行版本，但存在大量警告”。

### HAPPY_END ——「稳定而温和的算界」

- 条件：
  - 主线完成，两条支线中至少一条完成度较高。
  - 在关键节点中多次选择“平衡技术与人性”的选项。
- 表现：
  - 算界各时代之间保持健康的技术传承，
  - 技术成果被尽可能用于帮助多数人。
  - 主角可以选择“留在某个时代”或者“继续作为观察者”。

### TRUE_END ——「超越单一时间线」

- 条件（综合）：
  - 主线全部完成。
  - 编程语言与操作系统两条支线完成度都很高。
  - 在多个关键抉择中选择了“责任、协作、开放性”的方向。
- 表现：
  - 主角在最终战中使用“元语言重构”能力，不是击败黑盒神谕，而是**重构其目标函数**：
    - 不再只是优化效率或收益，而是引入“可解释性、公平性、人类福祉”等约束。
  - 算界不再是被动执行历史的世界，而是获得持续自我反思与调整的能力。
  - 时间编译器将权限部分移交给主角，提示“你已不再只是调试者，而是共创者”。

## 六、后续可扩展方向

- 为每个时代增加更多现实工程产物对应的装备树（如：路由器 → 网络斗篷，SSD → 闪存之靴等）。
- 细化每章的支线人物（日常对话、好感度系统）。
- 设计具体的事件分支，让玩家的技术选择真实影响战斗和剧情。
